My implementation attempt of a chess bot in C.

In the pursuit of learning memory usage in C and other low-level languages I decided to take on the task of building a chess bot algorithm from basically scratch on my own. My motivation for this was not only my love for the game but also the idea of compressing it as much as I could. This project is still ongoing, and I will be updating this at some points perhaps.

Before I actually can begin building the bot, it is necessary to actually make a board be somehow represented in the program, so I decided it would be wise to compress it as much as possible. I decided to represent the pieces as 4-bit numbers, in rows of 32-bit integers, the default Int size in C. This would result to the following representation of the intial chess position:

board[7] = 0x3759B573;
board[6] = 0x11111111;
board[5] = 0x00000000;
board[4] = 0x00000000;
board[3] = 0x00000000;
board[2] = 0x00000000;
board[1] = 0x22222222;
board[0] = 0x486AC684;

Moving the position into a 16-bit system makes it easier to recognise which numbers represent which pieces, making the retrieval of a piece obvious in the following function:

unsigned int pieceAt(int col, int row)
{ // pos = 1 to 8
return (unsigned int)(*board + (row - 1)) << (4 * (col - 1)) >> 28;
}
